ENGLISH
03/11/2024
In order to have better organization in the files and also
to avoid a lot of rework as the application grows, I will
refactor the routes section so that this vital part of the
application is separated from the rest.

This way, besides saving time in adding new routes and
testing existing ones, the code becomes cleaner and more
readable.

As I am currently working with 'users', I will create the
routes file related to 'users', with the methods we already
have today...Remember, throughout the refactoring process, I
will have 'secure-mode' enabled to have instant feedback on my
changes.

Note that I made a drastic change in the architecture of my
application, refactored codes, and created new files... If I
were to test all the altered parts, it would take valuable
development time, but since I have all the tests prepared and
work in secure mode, as soon as I saved my changes, I had feedback
in an AMAZING '0.965' SECONDS!!!

Here we see the importance of having automated tests that clearly
cover the provided scenarios... Of course, we still have a few
routes with simple functionalities, but I am confident that whatever
I do from now on, my tests will tell me if it is correct or not!
